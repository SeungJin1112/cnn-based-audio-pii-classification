# CNN-Based Audio PII Classification

## 개요

이 프로젝트는 음성 데이터셋을 직접 생성한 뒤, 해당 음성 자체의 음향적 특징을 기반으로 CNN 분류를 수행하여 STT(Speech-to-Text) 과정 없이 민감정보 포함 여부를 분류하는 것을 목표로 합니다.

현재 저장소에는 전화번호가 포함된 한국어 음성 샘플과 메타데이터가 생성되어 있으며, 이를 기반으로 이후 스펙트로그램 변환, CNN 학습, 추론 파이프라인을 구성할 수 있도록 준비된 상태입니다.

## 목표

- 음성에서 텍스트를 먼저 복원하지 않고 민감정보 포함 여부를 분류
- 전화번호와 같은 개인정보 패턴이 포함된 발화를 음향 특징만으로 탐지
- STT 의존도를 줄여 처리 시간, 비용, 개인정보 노출 가능성을 낮추는 구조를 지향

## 현재 생성된 데이터셋

현재 확인된 산출물 기준으로 다음 데이터가 존재합니다.

- `data/00000/output` 하위에 전화번호 포함 음성 `.wav` 파일 100개
- `data/00000/metadata.csv` 메타데이터 100건
- 음성 포맷: `16kHz`, `mono`, `wav`
- 화자 유형: 남성 / 여성 TTS 음성 혼합
- 발화 내용: 전화번호를 다양한 템플릿 문장에 삽입한 한국어 문장

참고: 현재 체크인된 데이터셋 스냅샷은 `data/00000` 아래의 100개 샘플 기준이며, `src/generate/generate.py` 내부 기본 설정값(`INPUT_NUMBER = "00001"`, `TOTAL_COUNT = 1000`)과는 차이가 있습니다.

예시 문장 패턴:

- `제 전화번호는 010-XXXX-XXXX입니다.`
- `연락처는 010-XXXX-XXXX입니다.`
- `문자 주실 번호는 010-XXXX-XXXX입니다.`
- `전화는 010-XXXX-XXXX으로 주시면 됩니다.`

즉, 현재 데이터셋은 `전화번호가 포함된 민감 발화` 클래스를 구성하기 위한 초기 데이터로 볼 수 있습니다.

## 메타데이터 구성

`metadata.csv`는 각 음성 파일에 대해 다음 정보를 기록합니다.

- `filename`: 음성 파일명
- `filepath`: 저장 경로
- `text`: 생성에 사용된 전체 발화 문장
- `phone`: 삽입된 전화번호
- `gender`: 화자 성별
- `voice`: 사용한 TTS 보이스 이름

예시:

```csv
filename,filepath,text,phone,gender,voice
phone_00000_male.wav,output\phone_00000_male.wav,번호 알려드릴게요. 010-3982-3343입니다.,010-3982-3343,male,ko-KR-InJoonNeural
```

## 데이터 생성 방식

`src/generate/generate.py`는 Microsoft Edge TTS와 ffmpeg를 사용해 데이터셋을 생성합니다.

생성 흐름은 다음과 같습니다.

1. 전화번호 형식의 가짜 개인정보 생성
2. 준비된 한국어 템플릿 문장에 전화번호 삽입
3. 남성 또는 여성 한국어 TTS 보이스 랜덤 선택
4. TTS 결과를 mp3로 생성
5. ffmpeg로 `16kHz mono wav` 변환
6. 메타데이터 CSV 저장

사용 보이스:

- `ko-KR-InJoonNeural`
- `ko-KR-SunHiNeural`

## 왜 STT 없이 CNN 분류를 하려는가

일반적인 접근은 다음 순서를 따릅니다.

1. 음성을 STT로 텍스트 변환
2. 텍스트에서 전화번호, 계좌번호, 주민번호 같은 민감정보 패턴 탐지

하지만 이 프로젝트는 이 과정을 거치지 않고, 음성 신호를 이미지형 특징으로 변환한 뒤 CNN이 직접 분류하도록 설계하는 방향을 목표로 합니다.

기대 효과는 다음과 같습니다.

- STT 오인식에 덜 의존하는 탐지 파이프라인 구성 가능
- 실시간 처리 시 계산 흐름 단순화 가능
- 민감 텍스트 자체를 중간 산출물로 남기지 않는 구조 설계 가능

## 권장 학습 파이프라인

현재 저장소에는 아직 `src/cnn` 내부 학습 코드가 없으므로, 아래와 같은 흐름으로 확장하는 것이 적절합니다.

1. `.wav` 파일을 Mel Spectrogram 또는 MFCC로 변환
2. 각 샘플에 라벨 부여
3. CNN 입력 형태로 정규화
4. 학습 / 검증 / 테스트 데이터 분리
5. CNN 분류 모델 학습
6. 추론 시 입력 음성이 민감정보 포함 음성인지 분류

예시 라벨 설계:

- `1`: 전화번호 등 민감정보 포함 음성
- `0`: 일반 발화 음성

현재 저장소에는 민감정보 포함 샘플만 존재하므로, 실제 CNN 학습을 위해서는 반드시 비민감 일반 발화 데이터도 추가해야 합니다.

## 현재 상태 요약

- 전화번호가 포함된 한국어 음성 데이터셋이 생성되어 있음
- 메타데이터를 통해 각 샘플의 텍스트, 전화번호, 화자 정보를 추적 가능
- CNN 학습용 디렉터리는 존재하지만 학습 코드는 아직 비어 있음
- 따라서 현재 단계는 `데이터 생성 완료 및 학습 준비 단계`로 보는 것이 정확함

## 실행 환경

데이터 생성 스크립트 기준 필요 환경:

- Python 3.8+
- ffmpeg
- `edge-tts`

설치 예시:

```bash
pip install edge-tts
```

ffmpeg는 별도 설치 후 시스템 경로에 등록되어 있어야 합니다.

## 핵심 정리

이 저장소는 현재 `전화번호가 포함된 민감 발화 음성 데이터셋`을 생성한 상태이며, 최종 목표는 이 결과물을 기반으로 CNN 분류기를 학습시켜 STT 없이 민감정보 포함 여부를 판별하는 것입니다.
